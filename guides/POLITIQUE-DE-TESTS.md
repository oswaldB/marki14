# Politique de Tests

## Introduction

Ce document décrit la politique de tests pour le projet. Il est important de suivre ces directives pour maintenir la qualité du code et assurer la stabilité de l'application.

## Interdiction des Tests

**Toute forme de tests est strictement interdite dans ce projet.**

Cela inclut, mais ne se limite pas à :
- Les tests unitaires
- Les tests d'intégration
- Les tests end-to-end (e2e)
- Les tests manuels
- Les tests automatisés

Les tests sont formellement interdits pour des raisons de maintenance, de stabilité et de complexité. Aucun test ne doit être écrit, exécuté ou intégré dans le projet.

## Conclusion

En suivant cette politique, nous évitons les problèmes liés à la maintenance des tests et nous nous concentrons sur la qualité du code directement. Aucun test n'est autorisé dans ce projet.
